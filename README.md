# Тестовое задание на примере игры Army Clash (Автобатллер)

## Архитектура
### Компонентная архитектура
Компонентная система с возможностью собирать героев как конструктор из ScriptableObject.
Каждый герой - набор независимых поведенческих и модифицирующих компонентов.

Используем DI (VContainer) + реактивный подход (UniRx).
Это полностью убирает жёсткие связи и отделяет логику от представления.

### UI
MVP-паттерн.
Презентеры не знают о конкретной визуализации — легко менять или добавлять новые виды интерфейса без правок бизнес-логики.

## Дополнительная фича
### Визуализация урона
Реактивно ловим событие урона → генерируем текст → вырезаем символы шейдером → формируем Custom Data → отправляем в Particle System.
Цифры красиво всплывают над персонажем.

Выбор именно данного функционала заключается в оживлении игры, а также для информативности.
Какие-либо доработки боя, героев - это было бы просто увеличение количества компонентов, 
а именно эта система показывает, как быстро и чисто добавить выразительную механику, не трогая ядро боя и не ломая существующие компоненты.
